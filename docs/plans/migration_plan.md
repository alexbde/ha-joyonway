# Migration Plan: ha-joyonway-p25b85 → ha-joyonway

> Move all content to a fresh repo `alexbde/ha-joyonway` with clean squash-merged PRs on `main`.

## Strategy

1. Create fresh repo `alexbde/ha-joyonway` on GitHub (no fork relation).
2. Push the upstream fork snapshot as the initial commit on `main`.
3. Apply the remaining changes in 7 logical PRs (replay non-merge commits by range onto feature branches, squash-merge each).
4. After all PRs are merged, rename the integration domain (`joyonway_p25b85` → `joyonway`) in a final PR.

This migration is **content-only replay**: exact commit SHAs/authorship/timestamps are not preserved on the new repo's `main` branch.

Original commit history remains visible in GitHub PR commit views from the old repo references. Keeping migration branches in the new repo is **not required**; they may be deleted after squash merge. The `main` branch stays clean (one squash commit per PR).

During migration, treat this old repo as **read-only**. Do not add new commits here unless the new repo (`alexbde/ha-joyonway`) is fully created, pushed, and ready to receive replay branches.

## Content Cleanup Rules

Use this migration as the opportunity to clean up the repo. Apply these rules
to **every commit** that lands in the new repo (fix during cherry-pick or in a
follow-up fixup commit per PR branch, before squash-merge).

### No capture data files

Remove all JSONL/JSON capture session logs from the tree. These contain
timestamps, real IPs, and local paths. They are bulky and not needed for the
integration itself — the protocol findings they produced are already documented
in `docs/protocol.md`.

**Files to exclude (currently tracked):**

| File | Why remove |
|------|-----------|
| `tools/captures_heating/heating_cycle_*.jsonl` | Timestamps, raw frames |
| `tools/captures_heating/heating_cycle_*.summary.txt` | Timestamps |
| `tools/captures_schedule_both/session_*.jsonl` | Real IP `192.168.x.x`, timestamps |
| `tools/captures_schedule_changes/session_*.jsonl` | Real IP, timestamps |
| `tools/captures_schedule_slot2/capture_slot2_*.jsonl` | Real IP, timestamps |
| `tools/captures_schedule_test/slot_test_*.jsonl` | Real IP, timestamps, local paths |
| `tools/captures_temp/temp_commands.json` | Temperature lookup (already in adapter code) |

**Action:** All capture data lives under `tools/captures/` (consolidated in
Step 3a) and is gitignored. Do not commit any capture data files. The capture
*scripts* (`tools/capture_*.py`, `tools/analyze_*.py`) are kept — they
document the methodology.

### No real IP addresses

The real bridge IP `192.168.x.x` must not appear anywhere in the new repo.

**Files requiring scrub:**

| File | Fix |
|------|-----|
| `tools/rebuild_manifest.py` | Replace hardcoded `192.168.x.x` with `192.168.1.100` (example IP) |
| `tools/captures_phase*/session_manifest.json` | These files are excluded (see above) |
| `tools/captures_crc/crc_session.json` | Excluded (see above) |
| `tools/captures_phase6/analysis_report.md` | Excluded (see above) |

**Dummy IPs are fine:** `192.168.1.100` (used as default in scripts via env
var fallback), `127.0.0.1` (used in tests), and example IPs in documentation
like `192.168.1.34` are all acceptable.

### No identifying timestamps in filenames or content

Capture filenames with `_YYYYMMDD_HHMMSS` suffixes embed session dates.
Since all capture data files are excluded (above), this is handled. For any
remaining files, strip timestamps from filenames and content.

**Exception:** Git commit metadata (author date, commit date) is fine —
that's standard git practice.

### No local filesystem paths

Some JSONL files contain `raw_bin_path` fields with full local paths like
`/Users/alex/IdeaProjects/...`. These files are all excluded (above).
Verify no remaining tracked file contains absolute local paths.

### No generated/build artifacts

**Exclude from new repo:**
- `ha_joyonway_p25b85.egg-info/` — generated by `pip install -e .`
- `*.egg-info/` already in `.gitignore`, but verify it's not tracked

### Proper `.gitignore`

The new repo's `.gitignore` should consolidate all exclusions. Replace the
old per-directory capture ignores with a single rule for the new layout:

```gitignore
# Capture session data (contains timestamps, IPs, local paths)
tools/captures/
*.jsonl

# Binary capture files
*.bin
```

This replaces the old `captures_phase4/`, `captures_phase5/`, etc. entries.

**Keep in `.gitignore`:** `.env`, `.local/`, `__pycache__/`,
`*.egg-info/`, `.venv*/`, build artifacts, IDE files,
`tests/live/artifacts_*/`.

### Images are fine

PNG icons (`icon.png`, `icon@2x.png`, `dashboard.png`) are acceptable —
they are required for HACS/branding and documentation.

### Summary checklist (per PR)

Before pushing each PR branch, verify:
- [ ] No `192.168.x.x` in any file (`grep -r '192\.168\.x' .`)
- [ ] No capture data files tracked (`git ls-files | grep -E 'captures/|\.jsonl|\.bin'`)
- [ ] No absolute local paths (`grep -r '/Users/' .`)
- [ ] No `*.egg-info` directories tracked
- [ ] `.gitignore` covers `tools/captures/`, `*.bin`, `*.jsonl`, `.env`, `.local/`

## Notes

- **Replay conflicts** are expected between PRs since later commits modify the same files. Each PR branch should be based on the previous PR's squash-merged `main`. Do them sequentially.
- **Merge commits:** replay non-merge commits only (for example with `git rev-list --reverse --no-merges <range> | xargs git cherry-pick`).
- **Alternative replay method:** Use `git format-patch` + `git am`:
  ```bash
  # In old repo, generate patches:
  git format-patch --no-merges 32e4e9a..b344095 -o /tmp/patches-pr1
  # In new repo, apply:
  git am /tmp/patches-pr1/*.patch
  ```
- **Content cleanup:** After cherry-picking each PR range, run the checklist from the **Content Cleanup Rules** section. Fix issues via `git commit --amend` (last commit) or `git rebase -i` (earlier commits in the branch) before pushing. This keeps the squash-merge result clean.
- **Tools directory** (`tools/`) contains capture *scripts* (`.py`) and analysis tools — these are kept. Capture *data* files (`.jsonl`, `.json` session logs, `.bin`) are excluded via `.gitignore` and must not be committed.
- **Tests** should pass at each PR boundary (or at least at PR 7). Minor test breakage in intermediate PRs is acceptable since this is content-only replay.
- **History visibility:** commit-by-commit history is expected to be reviewed from GitHub PR commit tabs; migration branches can be deleted after merge.
- The domain rename (PR 8) is the only PR that modifies file paths — keep it separate and last for clean review.
